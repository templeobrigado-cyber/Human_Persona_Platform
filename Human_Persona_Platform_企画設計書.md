# Human Persona Platform（HPP）企画・設計書

## 1. ビジョン

### ミッション

人の「性格」「価値観」「行動特性」「動機」をデータ化し、あらゆるサービスとの最適なマッチングを実現する。

``` text

 ↓
人格データ生成
 ↓
Persona ID発行
 ↓
AI分析
 ↓
業界別マッチング
```

## 1.5 差別化戦略：二枚看板モデル

診断エンジンの核（タイプコンパス＋Big Five＋エニアグラム）は共有しつつ、**出力の見せ方を市場ごとに分岐**させる。これにより1つのデータ基盤で相反する2つの信頼獲得軸（占い＝感情訴求／企業＝科学的根拠）を同時に満たす。

  市場           コアエンジン                              追加レイヤー   出力トーン                   訴求軸
  -------------- ----------------------------------------- -------------- ---------------------------- --------------------------
  占い師・BtoC   タイプコンパス＋Big Five＋エニアグラム     九星気学       運命・宿命の物語として語る   親しみやすさ・当たる感
  採用・企業     タイプコンパス＋Big Five＋エニアグラム     なし           心理統計に基づく分析として語る   客観性・説明責任

-   九星気学は生年月日から一意に算出できるため、**入力負荷を増やさずに占い版だけ「当たる感」を追加**できる。
-   企業向けには九星気学を混ぜない。人事・採用の意思決定に「占い」を持ち込むと説明責任が果たせず信頼を損なうため、あえて外す。
-   同じPersona Engineから2つのSKUが作れるので、開発コストは1つ、マーケットは2つという構造。

### ネーミング：「タイプコンパス」

ユング心理学ベースの4指標（E/I・S/N・T/F・J/P）診断は、ユーザー向けの名称を**「タイプコンパス」**とする。

-   **理由**：「MBTI」という名称自体はThe Myers-Briggs Companyの商標であり、公式設問を使わず独自設問で診断を作っても、「MBTI診断」と名乗って商用展開するのは商標リスクがある。
-   **メタファー**：4つの軸＝方角（東西南北）で自分のタイプの「方角」を示す、というコンパスのイメージ。直感的でキャッチーな一方、法的リスクを避けられる。
-   **使い分け**：ユーザーが見る画面・マーケティング・商品名では「タイプコンパス」を使う。設計書内でロジックの元になる心理学理論を説明する箇所（本ドキュメント内の技術セクション）でのみ「ユング理論ベースの4指標（旧称：MBTI理論）」と補足する。DBのテーブル名・カラム名（`mbti_results`等）は内部の技術識別子なのでリネーム不要。

## 2. 人格データモデル

  レイヤー   診断           目的               適用対象
  ---------- -------------- ------------------ ------------------
  L1         タイプコンパス 行動傾向           共通
  L2         Big Five       性格特性           共通
  L3         エニアグラム   行動動機           共通
  L4         九星気学       宿命・運勢傾向     占い版のみ
  L5         価値観診断     人生観・仕事観     共通
  L6         適性診断       能力分析           企業版のみ
  L7         行動分析AI     嗜好・習慣         共通

``` json
{
  "persona_id":"P-000001",
  "creativity":92,
  "leadership":75,

  "challenge":95,
  "stability":21,
  "communication":68
```
