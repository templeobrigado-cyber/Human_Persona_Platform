# DB完全設計

## users

id(PK) name email password birth_date

> birth_date は九星気学（本命星・月命星）の算出に使う。占い版を利用しない企業ユーザーも入力自体は任意でよい。

## diagnosis_sessions

id(PK) user_id(FK) status created_at

## questions

id category question weight

## answers

id session_id question_id answer

## mbti_results（ユーザー向け表示名：タイプコンパス）

user_id type ei sn tf jp

> テーブル名は内部技術識別子としてそのまま維持する。画面・API応答・AI出力ではユーザー向け名称「タイプコンパス」に変換して表示すること。

## bigfive_results

user_id openness conscientiousness extraversion agreeableness
neuroticism

## enneagram_results

user_id type wing

## kyusei_results（占い版のみ）

id(PK) user_id(FK) honmeisei（本命星） getsumeisei（月命星） gogyou（五行）

## persona_profiles（共通・企業版で使用）

persona_id user_id creativity leadership empathy stability challenge
logic communication discipline

## persona_profiles_fortune（占い版のみ）

persona_id user_id creativity leadership empathy stability challenge
logic communication discipline gogyou_bonus（五行による加点内訳）

## compatibility

id persona_a persona_b score analysis variant（standard／fortune）

## payments

id user_id product price

## ER

``` text
users
 ├ diagnosis_sessions
 │    └ answers
 ├ mbti_results
 ├ bigfive_results
 ├ enneagram_results
 ├ kyusei_results（占い版のみ）
 ├ payments
 ├ persona_profiles（共通・企業版）
 │    └ compatibility（variant=standard）
 └ persona_profiles_fortune（占い版のみ）
      └ compatibility（variant=fortune）
```
