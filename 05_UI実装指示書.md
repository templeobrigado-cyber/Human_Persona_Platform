## 1. スコープ

### 今回作るもの（MVP）
1. **TOP**（`/`）— 占い版／企業版の2つの入り口
2. **属性入力**（占い版のみ生年月日入力）
3. **診断画面** — 81問をステップ分割、進捗表示
4. **結果画面** — 占い版／企業版で表示を分岐

### 今回作らないもの
- DB保存・認証・決済（購入ボタンはダミーCTA。`onClick`なしの見た目だけで良い）
- AI鑑定・AI分析の生成（結果画面にプレースホルダー枠のみ用意）
- マイページ・占い師向け画面・管理画面
- ダークモード（**ライト固定**。`globals.css`の`prefers-color-scheme: dark`ブロックは削除してよい）

### 診断結果の保持
DBがないため、回答と結果は**クライアント側で完結**させる：
- 回答状態はReact state（`useState`/`useReducer`）
- 結果画面への受け渡しは`sessionStorage`（キー例：`hpp:result:fortune`）。リロード時にsessionStorageから復元、なければTOPへリダイレクト

## 2. 技術前提（重要・ハマりどころ）

### Next.js 16.2.10（App Router）
`node_modules/next/dist/docs/` に同梱ドキュメントあり。**書く前に該当ガイドを読むこと**。特に：
- **`params`/`searchParams`はPromise**。同期アクセスは16で完全廃止。`const { variant } = await params`（Server Component）または`use(params)`（Client Component）
- `next lint`は廃止済み。lintは`npx eslint`を直接使う（`npm run lint`が`eslint`を指している）
- `middleware.ts`は`proxy.ts`に改名済み（今回は使わない）
- Client Componentは必要最小限に。`'use client'`はインタラクティブな末端コンポーネントに付け、ページ全体をClient化しない（診断画面は回答stateが必要なのでClient化やむなし。TOPと結果画面の枠はServerのまま）

### Tailwind CSS v4
- 設定ファイル（`tailwind.config.js`）は**存在しない**。テーマは`src/app/globals.css`の`@theme`ブロックにCSS変数で定義する（CSS-first設定）
- `@theme`に`--color-xxx: ...`を定義すると`bg-xxx` `text-xxx`等のユーティリティが自動生成される

### その他
- React 19 / TypeScript strict。`any`禁止（eslintで落ちる）
- 追加npmパッケージは**入れない**。レーダーチャートは手書きSVGで実装（§5.7）
- 検証は `npm run build` / `npm test` / `npx eslint src/` の3点セット

## 3. 診断ロジックAPI（実装済み・そのまま使う）

```ts
// src/lib/type-compass.ts — 24問
TYPE_COMPASS_QUESTIONS: { id, axis, pole, text }[]
diagnoseTypeCompass(answers: Record<string, 1|2|3|4|5>): TypeCompassResult
// → { type: "ESTJ"等, axisResults: {axis, dominant, strength}[], traitVector: {...8値} }

// src/lib/big-five.ts — 30問（逆転項目あり。逆転処理はロジック側で済むのでUIは素点を渡すだけ）
BIG_FIVE_QUESTIONS: { id, factor, reversed, text }[]
diagnoseBigFive(answers): BigFiveResult
// → { scores: {openness, conscientiousness, extraversion, agreeableness, neuroticism}, traitVector }

// src/lib/enneagram.ts — 27問
ENNEAGRAM_QUESTIONS: { id, type, text }[]
ENNEAGRAM_TYPE_NAMES: Record<1..9, string>  // "改革する人"等
diagnoseEnneagram(answers): EnneagramResult  // → { type, wing, typeScores }

// src/lib/kyusei.ts — 設問なし、生年月日から算出（占い版のみ）
diagnoseKyusei({ year, month, day }): KyuseiResult  // → { honmeisei, honmeiseiName, gogyou }
// 不正な日付はthrowするのでUI側でバリデーション or try-catch

// src/lib/persona-engine.ts — 統合
buildPersonaProfile({ typeCompass, bigFive, enneagram }): PersonaProfile          // 企業版
buildPersonaProfileFortune({ ...同上, kyusei }): PersonaProfileFortune           // 占い版
PERSONA_AXES: PersonaAxis[]  // 8軸のキー配列
// 全診断関数は未回答があるとthrow → UI側で全問回答済みを保証してから呼ぶ
```

**設問数**: タイプコンパス24＋Big Five 30＋エニアグラム27＝**81問**。全問5件法Likert（1〜5）。

## 4. ルーティングと画面フロー

```
/                                TOP（Server Component）
/diagnosis/[variant]/start       属性入力（variant = "fortune" | "business"）
/diagnosis/[variant]/questions   診断画面（Client）
/diagnosis/[variant]/result      結果画面
```

- `variant`のバリデーション：`fortune`/`business`以外は`notFound()`
- フロー：TOP →（占い版のみ）start で生年月日入力 → questions → 全問回答で診断実行＆sessionStorage保存 → result
- 企業版のstartは「所要時間・設問数の説明＋開始ボタン」のみ（生年月日不要）

### 画面別仕様

#### TOP `/`
ワイヤーフレーム準拠：ロゴ／キャッチコピー／CTA2つ（占い版・企業版）／特徴3点／料金プレースホルダー。
- CTAは「入り口で迷わせない」が最重要。占い版=紫系・企業版=青系で**色でも区別**する（§5.2）
- キャッチコピー案：「4つの理論で、あなたの人格を1枚に。」（変更可）
- 「81問・約10分・無料」を明記

#### 属性入力 `/diagnosis/fortune/start`
- 生年月日入力：**`<select>`3つ（年・月・日）**を推奨（スマホでのdate picker差異を回避）。年は1930〜今年
- 「生年月日から九星気学の本命星を自動算出します」の説明文
- 入力値はsessionStorageに保存して questions へ

#### 診断画面 `/diagnosis/[variant]/questions`
- **6問/ページ × 14ページ**（最終ページは3問）。セクション順：タイプコンパス24→Big Five 30→エニアグラム27
- セクション切替時に小さな中間演出（「Part 2/3 ここから行動スタイルについて」等）で疲労を軽減
- 進捗表示：上部に**プログレスバー＋「n / 81」**
- Likert UI：5つの丸ボタン（左「そう思わない」右「そう思う」のラベル）。**radioグループとして実装**（`role="radiogroup"`、キーボード操作可）
- ページ内全問回答で「次へ」が活性化。「戻る」で前ページへ（回答は保持）
- 回答stateは`useReducer`推奨。ページ遷移で失わないよう questions ページ内はステップをstateで管理（URLは変えない）
- 全81問回答 → `diagnose*`3つ＋（占い版は`diagnoseKyusei`）→ `buildPersonaProfile(Fortune)` → sessionStorage保存 → resultへ`router.push`

#### 結果画面 `/diagnosis/[variant]/result`
表示順（ワイヤーフレーム準拠）：

| セクション | 占い版 | 企業版 |
|---|---|---|
| タイプコンパス | ✅ 4文字タイプ＋軸バー | ✅ 同左 |
| Big Fiveレーダー | ✅ 5角形 | ✅ 5角形 |
| エニアグラム | —（非表示） | ✅ タイプ名＋ウイング |
| 九星気学 | ✅ 本命星・五行カード | **❌ 絶対に表示しない** |
| Personaベクトル | ✅ 8軸 | ✅ 8軸 |
| AI鑑定/AI分析 | プレースホルダー（運命トーン枠） | プレースホルダー（心理統計トーン枠） |
| 購入CTA | ✅ ダミー | ✅ ダミー |

- 「もう一度診断する」リンクをフッターに

## 5. デザインシステム

### 5.1 トーン
- **共通**: 清潔・信頼感・データの見える化。フラットで余白多め、影は控えめ（`shadow-sm`まで）
- **占い版**: 神秘性を「上品に」。深い紫＋金のアクセント。ギラギラした占いサイト風にしない
- **企業版**: HRテック風。青＋スレート。グラフと数値を主役に

### 5.2 カラー（`globals.css`の`@theme`に定義する）

```css
@theme {
  /* 共通ニュートラル */
  --color-surface: #fafaf9;        /* ページ背景 */
  --color-card: #ffffff;
  --color-ink: #1c1917;            /* 本文 */
  --color-ink-muted: #78716c;      /* 補足テキスト */
  --color-line: #e7e5e4;           /* 罫線 */

  /* 占い版（fortune） */
  --color-fortune: #6d28d9;        /* メイン: violet-700 */
  --color-fortune-deep: #4c1d95;   /* 見出し・ヒーロー背景 */
  --color-fortune-soft: #f5f3ff;   /* カード背景 */
  --color-fortune-accent: #b45309; /* 金アクセント（本命星など） */

  /* 企業版（business） */
  --color-business: #2563eb;       /* メイン: blue-600 */
  --color-business-deep: #1e3a8a;
  --color-business-soft: #eff6ff;

  /* チャート用 */
  --color-chart-fill: #8b5cf6;     /* fortune時 */
  --color-chart-fill-biz: #3b82f6; /* business時 */
}
```

- variant別の色はコンポーネントに`variant`プロップを渡して`bg-fortune`/`bg-business`を切り替える（`data-variant`属性＋CSS変数の上書きでも可。どちらかに統一）
- コントラスト：本文`ink`/背景`surface`はWCAG AA以上を維持。ボタン文字は白×メイン色

### 5.3 タイポグラフィ
- 日本語フォント：**Noto Sans JP**（`next/font/google`）に差し替え。`layout.tsx`のGeistを置き換え、`--font-sans`に割り当てる。数値・タイプ表記（ESTJ等）はGeist Monoを残して使うと締まる
- スケール：h1=`text-3xl/tight font-bold`、セクション見出し=`text-xl font-bold`、本文=`text-base`、補足=`text-sm text-ink-muted`
- `layout.tsx`の`lang`を`"ja"`に、`metadata`をサービス名に変更（title: 「HPP — 人格診断プラットフォーム」仮）

### 5.4 レイアウト
- モバイルファースト。コンテンツ最大幅：診断画面`max-w-xl`、TOP/結果`max-w-3xl`、中央寄せ
- カード：`rounded-2xl border border-line bg-card p-6`
- タップターゲット44px以上（Likertボタンは`size-11`以上）

### 5.5 用語ルール（絶対）
- 画面上の表記は**「タイプコンパス」**。「MBTI」という単語をUI文言・コード内のユーザー向け文字列に**一切出さない**（商標回避。内部変数名は自由）
- 企業版の画面・コードパスに九星気学データを渡さない（Persona Engineの2 SKU分岐と同じ原則）

### 5.6 コンポーネント構成（`src/components/`）

| コンポーネント | 役割 |
|---|---|
| `Button` | variant色対応のprimary/secondary |
| `Card` | セクション枠 |
| `ProgressBar` | 診断進捗（%幅の内側バー） |
| `LikertScale` | 1問分の5択radio。左右にラベル |
| `AxisBar` | タイプコンパスの軸表示（E ◀━━●━━▶ I、strength%） |
| `RadarChart` | Big Five 5角形（§5.7） |
| `PersonaRadar` or `PersonaBars` | Personaベクトル8軸（レーダー8角形 or 横棒。実装しやすい方でよいが8軸レーダー推奨） |
| `KyuseiCard` | 本命星名・五行を金アクセントで表示（占い版のみ） |

### 5.7 レーダーチャート実装方針（ライブラリ禁止）
純SVGで実装。要点：
- `viewBox="0 0 300 300"`固定、中心(150,150)、半径110
- 頂点座標：`angle = -90° + i * (360°/n)`、`x = cx + r * (score/100) * cos(angle)`
- 背景に20/40/60/80/100%のグリッド多角形（`stroke-line`）＋各軸ラベル
- データ多角形は`fill`をチャート色の30%透過、`stroke`を実線
- Server Componentで描画可能（インタラクション不要）。`role="img"`と`aria-label`でスコアを読み上げ可能に

## 6. 実装の進め方と検証

1. `globals.css`テーマ＋`layout.tsx`（フォント・metadata・lang）→ 2. 共通コンポーネント → 3. TOP → 4. start → 5. questions → 6. result の順で、**画面ごとに`npm run build`を通しながら**進める
2. 診断実行部分はロジックのテストが既にあるためUIテストは必須ではないが、**回答state管理（reducer）を関数として切り出す場合はVitestを足す**
3. 完了条件：
   - [ ] `npm run build` / `npm test` / `npx eslint src/` すべてクリーン
   - [ ] 占い版：TOP→生年月日→81問→結果（九星カードあり）が通しで動く
   - [ ] 企業版：TOP→開始→81問→結果（九星なし・エニアグラムあり）が通しで動く
   - [ ] 未回答ページでは「次へ」が押せない／結果画面直リンクはTOPへ戻る
   - [ ] モバイル幅（375px）で崩れない
   - [ ] 画面内に「MBTI」表記がない
4. 作業後に`WORKLOG.md`へ記録を追加（このリポジトリのルール）
